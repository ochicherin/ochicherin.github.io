When using Oracle Text component AUTO_LEXER is buggy
highlighting words. Using AUTO_LEXER critical, because only
It is supported by Russian morphology.
Incorrect results are the procedures
CTX_DOC.HIGHLIGHT,
CTX_DOC.MARKUP,
CTX_DOC.TOKENS,
CTX_DOC.SNIPPET
responsible for highlighting the results of full-text query.

The problem is that Oracle Text misinterprets
Russian letters "Ё" and "Й" as dual characters.
As a result, unless the text there are the symbols,
we see the effect of shifting lights, growing at each occurrence
letters "Ё" and "Й".

Consider an example.

Create a test table.
Let's fill it with data.
And construct a full-text index.


More spotlight text says with ctx_doc.snippet


The first and second requests have worked correctly.
In the third query incorrectly determine the length of the word because 
of the presence of the symbol "Й".
In the fourth Inquiry incorrectly identified the initial position of word.

Let us make a more systematic test.
The test line contains all the letters of the Russian alphabet.
We will highlight them in turn. This will reveal all the letters that influence
correct illumination.


As seen from the example. Starting with the letter "А" to the letter "Е" and lighting works properly.
The letter "Ё" is defined as a double symbol.
Next to the letters "Й", the observed shift lights for 1 symbol.
The letter "Й" is defined as a dual symbol.
Next, observe the shift lights to 2 characters.

This bug makes it virtually impossible to use backlighting applications.
